### PROMPT 11/25 — Payments Module (MoMo + VNPay)
**File tham chiếu:** `10-to-17-modules-advanced.md` (section "Module Payments")
**Thời gian ước tính:** 8 phút

```
Đọc docs/specs/10-to-17-modules-advanced.md section "Module Payments".

1. src/RentalOS.Infrastructure/Services/Payments/MoMoService.cs:
- CreatePaymentAsync(invoice, returnUrl, notifyUrl):
  Build request đúng MoMo API v2 (requestId, orderId format: {tenantSlug}-{invoiceCode}-{timestamp})
  Sign: HMAC-SHA256 rawSignString theo MoMo spec
  POST https://payment.momo.vn/v2/gateway/api/create
  Return CreatePaymentResult { PayUrl, QrCodeUrl, Deeplink }

- ProcessWebhookAsync(rawBody, headers):
  1. Tìm tenant từ orderId prefix
  2. Verify HMAC-SHA256 signature
  3. Redis idempotency: SET NX key "payment:idempotent:momo:{orderId}" TTL 72h
     Nếu key đã tồn tại → return WebhookResult { AlreadyProcessed: true }
  4. Nếu resultCode != 0 → log và return (không xử lý payment thất bại)
  5. Tìm invoice từ orderId
  6. Tạo Transaction record
  7. Update invoice.status = Paid, invoice.paid_at = NOW()
  8. Enqueue SendPaymentConfirmationJob
  9. Broadcast SignalR "InvoicePaid" event

2. src/RentalOS.Infrastructure/Services/Payments/VNPayService.cs:
- CreatePaymentAsync: build query string, sign HMAC-SHA512
- ProcessWebhookAsync: verify vnp_SecureHash
- ProcessReturnAsync: xử lý GET return URL
